# Rescue

This page explains what `blinx` preserves and how to separate the active BLINX line from inherited Conker history.

## What Was Rescued

- the inherited runtime surface still lives under [`../conker`](../conker/README.md)
- detailed BLINX and inherited Conker notes live under [`../conker/docs`](../conker/docs/README.md)
- early inherited Conker context is indexed in [`./archive/README.md`](./archive/README.md)
- invalidated inherited Conker branches are indexed in [`./branches/README.md`](./branches/README.md)

## Why It Was Rescued

- BLINX depends on the post-reset Conker substrate
- the inherited Conker legality lessons still explain the design constraints
- the older notes remain useful for chronology and falsification

## How To Read It

- treat `docs/` as the curated public BLINX layer
- treat `conker/docs/` as the dense note store
- treat inherited Conker notes as context, not as active BLINX branches

See also:

- [Salvage Matrix](../SALVAGE_MATRIX.md)

## What Is Not Fully Normalized

- deeper inherited docs still contain local-path references
- old matrices and handoffs still reflect lab-era artifact locations
- those files are preserved for rescue value, not rewritten as polished public docs
